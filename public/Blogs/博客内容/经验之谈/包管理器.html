包管理器一览

什么是包管理器

在阅读美化终端专题时，有一步3-2是下载Oh My Posh，讲的是如何纯手动下载、安装Oh My Posh。如果进一步阅读Oh My Posh 官方文档中的 安装：Windows一节，会发现官网提供了 4 种安装方式：

除了第 3 种方式：manual手动外，另外三种方式，都是通过包管理器进行安装。

包管理器已经成为现代编程开发非常重要的一种软件包管理手段，以统一的方式自动安装、升级、配置和删除计算机程序。包管理器旨在消除手动安装和更新的繁琐步骤，简化流程。这对于操作系统通常由数百甚至数万个不同的软件包组成的大型企业来说尤其有用。

举个例子：假设你想要用CMake来编译并运行C语言，那你至少要下载 3 个软件：

  • GCC 编译器
  • CMake
  • Ninja

每一个软件，都要你自己去找网站下载，再安装、配置，非常地麻烦。这时，用包管理器的优势就体现出来了，我们可以用pacman包管理器来统一安装这 3 个软件： 

      
        pacman -S mingw-w64-x86_64-gcc
        pacman -S mingw-w64-x86_64-ccmake
        pacman -S mingw-w64-x86_64-ninja

看出统一语法的优势了吗？你也不用管这些软件在哪个网站上，pacman会自动帮你搞定！正因为具备诸多的优势，包管理器已成为现代开发必不可少的工具。

常用包管理器简介

  • Chocolatey

    专为 Windows 打造的现代软件自动化包管理器。

    • 官网： https://chocolatey.org/
    • GitHub：https://github.com/chocolatey/choco
  • Scoop

    专为 Windows 打造的命令行安装器。

    • 官网：https://scoop.sh/
    • GitHub：https://github.com/ScoopInstaller/Scoop
  • winget

    Windows 程序包管理器是一个全面的程序包解决方案，由一个命令行工具winget和一组用于在 Windows 设备上安装应用程序的服务组成。

    • 官网：https://learn.microsoft.com/zh-cn/windows/package-manager/
    • GitHub：https://github.com/microsoft/winget-cli
  • npm

    专为 JavaScript 程序包打造的管理器，会随着Node.js一起安装。

    • 官网：https://www.npmjs.com/
    • GitHub：https://github.com/npm/cli
  • pacman

    在Linux中也可用于 Windows管理软件包的实用程序。

    • 官网：https://wiki.archlinux.org/title/pacman
    • GitLab：https://gitlab.archlinux.org/pacman/pacman

安装和使用Chocolatey

安装Chocolatey

在Chocolatey里，有两个页面都介绍了如何安装，二者的主要内容是一样的：

  • 安装页面
  • 文档页面的命令行界面安装专题

官网推荐使用PowerShell安装Chocolatey：

  1. 确保用管理员身份运行PowerShell

    在Windows 终端里用管理员身份运行PowerShell，会打开一个新的终端窗口，左上角有一个盾牌标志，表示当前终端是用管理员身份运行的。

  2. 查看PowerShell执行策略

    返回RemoteSigned。如果返回Restricted，就要运行第 3 步进行修改：

    小知识

    有关PowerShell执行策略的区别，请参阅PowerShell：关于执行策略。

  3. 修改PowerShell执行策略只要第 2 步返回的不是Restricted，就可以跳过第 3 步

    用PowerShell运行以下代码二选一即可： 

              
            Set-ExecutionPolicy AllSigned
            // 或者 ----------------------
            Set-ExecutionPolicy Bypass -Scope Process
  4. 执行安装命令 
              
            Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

    不用怀疑，这是官网提供的超长命令，直接复制运行，耐心等待安装过程。如果未显示任何错误，就表示安装成功。

  5. 运行Chocolatey自检命令

    可以用以下命令确认Chocolatey是否正确安装： 

              
            choco
            // 或者 ----
            choco --version

    不管哪个命令，只要弹出相关版本信息2.2.2，就表示安装成功。

安装Scoop

  2. 运行安装命令
