前端与后端

Web开发职责划分

  • Web前端：负责用户界面UI，直接面向用户；要求：美观、辨识度高、易用。
  • Web后端：负责数据处理数据库、文件，后台服务；要求：精准、严谨、高效。

小知识

只有 前端 的网站，称为静态网站。既有前端、又有后端的网站，称为动态网站。
静态网站，意味着不管谁浏览，都会看到同样的内容，同样的效果。
动态网站，意味着不同的用户，能够看到不同的内容，不同的效果。
比如：浏览淘宝网站时，经常买衣服的用户，会更频繁地看到服装类商品；经常买零食的用户，会更频繁地看到食品类商品。为什么会这样？就是因为淘宝在后端分析用户平时浏览的数据，再根据分析结果，推送相关的内容到前端。

Web 前端简介

Web 前端的核心

前端是一个充分发挥创造力和表现力的工作，现代前端的框架非常多，但其核心支柱就只有3个。

  • HTML 负责页面内容
  • CSS 负责页面样式
  • JavaScript 负责页面交互

永恒代码统计

提醒

为了提高开发效率，永恒项目也引用了一些第三方库它们都已从代码统计数据中剔除，比如：

  • 为了呈现图表效果，引用了Apache ECharts
  • 需要以格式化的形式展示代码，引用了Prism
  • 为了处理不同颜色格式之间的转换，引用了chroma.js
  • 需要更好的颜色拾取器，引用了Pickr
  • 需要在播放视频时获得更多自定义选项，引用了Video.js

这些库其实就是大量事先编写好的JavaScript和CSS代码。我们当然也可以自己纯手动编写来实现图表、代码格式化等等功能，但这样太花时间、太费精力。如果是闲暇有空学了玩玩还行，真正开发项目时，效率非常重要，要尽可能快地交付产品，不可能等着你慢慢开发基础功能。
