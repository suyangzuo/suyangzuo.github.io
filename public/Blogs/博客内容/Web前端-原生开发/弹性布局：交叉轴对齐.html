弹性布局：交叉轴对齐原创

作者： 苏扬

日期： 2024年6月29日

提醒

在弹性布局和网格布局中，交叉轴的含义有所不同；本专题讨论的是弹性布局中的交叉轴。

欢迎访问本站交互式学习中心的弹性布局：交叉轴对齐以获得更好的交互式体验。

交叉轴简介

什么是交叉轴？

主轴与弹性方向平行；交叉轴与弹性方向垂直。

弹性方向决定了主轴与交叉轴的方向。

提醒

本专题将以默认的弹性方向为基础，即主轴为行，交叉轴为列。

为什么要单独讨论交叉轴？

在交叉轴方向，有时会出现多个行；行本身、行内部的元素如何布局，分别对应align-content和align-items，这是很多人初学弹性布局最困惑的地方。因此，本专题将专门讨论交叉轴的对齐方式。

弹性如何体现？

一个元素如果使用了弹性布局，其内部的直接子元素会成为一个弹簧，意味着可以收缩和膨胀；区别在于：默认状态下会收缩而不会膨胀。

不管增加多少元素，它们都会尽可能地收缩在弹性容器内部，就像很多弹簧被压缩着挤在一个盒子里，这就是默认状态下会收缩的特性。

维持弹簧原本的尺寸

弹性布局内部的直接子元素默认会收缩。如果希望维持它们原本的尺寸，该怎么做呢？方法有两种：

  • 设置弹性收缩为0 
              
            弹性容器的直接子元素 {
              /* flex-shrink 默认为 1 */
              flex-shrink: 0;
            }

    设置了flex-shrink: 0后，弹性容器的直接子元素不会再收缩，而是维持原本的大小。不过，这种方式有时会导致子元素溢出容器。

  • 允许换行，一行放不下就放到下一行 
              
            弹性容器 {
              /* flex-wrap 默认为 nowrap */
              flex-wrap: wrap;
            }

    设置了flex-wrap: wrap后，一行放不下的元素，会自动放到下一行，这就是多行的由来。可以看出，这几行是从上往下生成的，意味着多行生成于交叉轴。

从以上方法可以看出，只有设置flex-wrap: wrap，交叉轴才会出现多行的情况。省略flex-wrap或者设置为flex-wrap: nowrap，交叉轴就只有一行。

交叉轴对齐

单行还是多行

让我们再次回顾并强调一下：

  • flex-wrap: nowrap：单行
    • align-items：有效
    • align-content：无效
  • flex-wrap: wrap：多行
    • align-items：有效
    • align-content：有效

单行行高

我们为一个正方形设置了800px × 800px的尺寸，并且设置其为弹性布局： 

      
        正方形 {
          display: flex;
          flex-wrap: nowrap; //此行写与不写效果相同，默认即为 nowrap
        }

由于flex-wrap: nowrap，因此整个方块内只有一行，align-content不起作用。这种情况下，一行会占满整个弹性容器的高度，即行高=800px。

align-content

align-content和align-items的区别，是很多人初学弹性布局时最困惑的地方。如果你仔细阅读了前面的内容，align-content的作用其实很简单：它控制所有的行而非行内的元素在弹性容器内如何对齐：

更改align-content，每一行的布局方式随之更改，但行内的元素相对于行的位置没有变化。我们可以隐藏所有的元素，更直观地感受一下什么是align-content：

align-items

align-items控制行内的元素在该行内如何对齐。

更改align-items，影响的是行内的元素，对行本身无效。

行内拉伸

仔细观察会发现，align-items也有normal和stretch选项：

align-items的默认模式也是normal，并且其效果也等同于stretch，这在MDN Web Docs中有明确的描述：

在normal的效果等同于stretch这一点上，align-content和align-items是相同的。但是，我们观察图15时，发现元素并没有被拉伸并填满整行。这是因为align-items: stretch的作用方式略有不同：

只有自动调整大小的元素，才会在行内拉伸。因此，要使元素高度拉伸，必须设置height: auto：

只有高度设置为auto的元素才会拉伸。换言之，align-items: stretch或normal必须和height或width: auto配合使用，方能达到拉伸一行内元素的效果。

提醒

事实上，弹性容器内的元素尺寸很少会设置为auto，因此，绝大部分情况下，align-items设置为normal、stretch、start，效果都是一样的。
