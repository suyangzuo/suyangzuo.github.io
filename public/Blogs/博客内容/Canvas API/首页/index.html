Canvas API简介

什么是Canvas API

引用MDN 文档中的原文：

简而言之，Canvas API的主要用途是：绘制2D图形。

为什么要用Canvas API

  1. 更好的性能

    Canvas绘制的所有图形，全部在Canvas元素内，不会产生任何新的DOM元素，因此，不会触发reflow，频繁的reflow会严重影响网页性能。因此，在处理大量图形时，Canvas的性能往往比DOM元素要好。

    例如：同学们可以访问交互式学习中心：变量。每次修改内存容量，所有的格子都会全部重新生成。我一开始就是用DOM元素来处理格子，能够明显感觉到页面的卡顿。后来我改用Canvas绘制格子，页面的流畅度就提升了很多。

    当然，Canvas API也有缺点。因为Canvas中都是图形，不是DOM元素，因此，无法使用事件侦听器进行绑定。类似鼠标进入一个矩形、鼠标点击一个圆，都要自己手动计算鼠标的位置和图形的位置。

  2. 精准的像素控制

    从一个圆上挖一个洞、把一个香蕉用抛物线扔出去……类似的动画，都可以用Canvas来精准实现。

Canvas API能做什么？

学习 Canvas API 后，你可以利用其强大的绘图和动画能力，在网页上实现高度定制化的图形渲染、交互设计和动态效果。以下是具体的应用场景和功能实现：

  • 数据可视化与图表

    用 Canvas 绘制实时更新的折线图、柱状图、饼图、散点图等，支持动画过渡和交互如悬停显示数据；渲染热力图、雷达图、树状图等传统图表库难以实现的图形。

  • 图像编辑与处理

    裁剪、旋转、缩放、滤镜；像素级操作马赛克、图像识别、图层混合；结合后端Node.js，批量生成海报、证书、商品主图，支持模板替换和数据填充。

  • 2D游戏开发

    绘制角色、场景、特效；通过键盘、鼠标或触摸事件实现游戏逻辑。

  • 交互式在线工具

    开发在线画板、涂鸦板，构建海报生成器、名片设计器。

  • 地图与地理信息

    渲染交互式地图热力图、路径规划，支持缩放和标记。
