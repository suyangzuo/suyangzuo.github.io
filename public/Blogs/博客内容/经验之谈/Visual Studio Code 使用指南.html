文本编辑器之争

文本编辑器

什么是文本编辑？文本编辑的对象是纯文本，因此Word不是文本编辑器，而是文档编辑器，因为Word的内容不仅包含文本，还包含了文本的格式如字体大小、颜色、段落间距等等。文本编辑器的目标，是在电脑上查看、运行；文档编辑器的目标，是打印到纸质材料上供人阅读。

代码也是文本，因此写代码也属于文本编辑。

我们最熟悉，也是最为简单的文本编辑器，是记事本，因为功能最少，因此启动速度也最快。文本编辑是最最常见的一种 IT 工作形式，主要的应用场景就是写代码、写文档。自从电脑发明以来，出现了无数的文本编辑器，包括了著名的编辑器之战。文本编辑器对于程序员来说，如同自己的左右手一样重要。

漫长的岁月中，无数的文本编辑器前赴后继。有已 30 岁高龄仍在坚持的UltraEdit；有不敌 VSCode，于2022年停止更新的Atom；有想努力挤进现代编辑器市场却以失败告终的Brackets；有性能优秀的Sublime Text；有专为 Vue 打造的HBuilder X；有水火不容的Emacs和Vim；更有新晋选手，号称 AI 优先的Cursor。还有更多文本编辑器无法一一列举，琳琅满目，让人目不暇接。

为什么选择Visual Studio Code

微软公司于 2015 年推出了Visual Studio Code简称VSCode，经过多年的完善、打磨，VSCode已经成为世界最受欢迎的文本编辑器。它完全免费、开源，启动速度快相对，扩展性强拥有海量扩展，支持多种流行语言，在 GitHub 上达到了惊人的 1560000 ！在YouTube和BiliBili上，绝大多数教程都是用VSCode作为编辑器的，可见其受欢迎程度。

正是因为VSCode的优秀和受欢迎，使得UltraEdit、Atom、Brackets、Sublime Text相形见拙，纷纷败下阵来。这些编辑器可能各有各的独特优点，但综合来讲，都不如VSCode。无论从性能、扩展性、颜值、免费、发展前景等角度来看，VSCode都是很长一段时间内的最佳选择。

Visual Studio Code基本设置

安装中文语言包

VSCode 初始语言为英语，其它语言是以扩展的形式存在的。因此我们需要打开扩展界面，安装简体中文包。VSCode 拥有海量扩展，一个个找非常麻烦，因此大部分情况下，我们都需要输入一些关键字，来过滤一下。

首先打开扩展界面，然后点击关键字搜索栏。

关键字输入chinese，经过过滤后，出现的扩展列表里第 1 个就是Chinese (Simplified) (简体中文) Language Pack for Visual Studio Code，点击Install开始安装。
安装完成后，右下角会弹出Change Language and Restart更改语言并重启，点击即可，重启之后，VSCode 的界面变成了简体中文。

自动保存

现代的文本编辑器大多具有自动保存功能，也就是说编写代码的同时，编辑器会自动进行保存，无需手动保存。这个功能确实很方便。

要启用自动保存，需要打开设置界面：

自动保存位于所有选项的第 1 个，可见其重要性和常用性。默认是off关闭，将其改成afterDelay一段延时之后，默认 1 秒即可。

提醒

如果在文件标签上看到一个白色圆点，表示此文件内容已更改但未保存。

这也意味着自动保存功能未启用。

有一种特殊情况：用 VSCode 新建的文件，在第 1 次保存之前，无论自动保存是否启用，都会出现白色圆点。

保存时格式化

我们平时写代码，在空格、缩进、符号等方面不会太讲究，导致代码格式不好看。这时候格式化功能就可以帮助我们自动进行格式化，不需要一个个手动去调。问题在于：什么时机进行格式化？

绝大多数情况下，我们都希望手动运行格式化功能，为其分配一个快捷键，想要格式化的时候按一下快捷键即可。另一种思路是：保存的时候进行格式化自动保存功能不算。这个功能在设置界面里，设置界面的选项非常多，与扩展一样，一个个找非常麻烦，我们同样需要用关键字来过滤。格式化的英文是format，因此在设置界面关键字搜索栏中，输入format，过滤一下。

过滤之后，点击文本编辑器的格式化选项卡，在右边的选项列表里，有一个Format On Save保存时格式化，勾选即可。

提醒

我个人是关闭保存时格式化功能的。因为 VSCode 有一个失去焦点时自动保存功能。想一下，什么时候会失去焦点？最常见的情况就是你切换了窗口。

切换窗口导致失去焦点，失去焦点导致自动保存，保存又导致格式化。连起来就成了：切换窗口时会格式化，因此你会发现，每次你切换窗口，再切回 VSCode，代码的格式都变了，这会破坏思维的连续性。因此，格式化功能还是手动触发比较好。

VSCode 格式化的默认快捷键是Alt + Shift + F。

修改字体

VSCode的字体设置需要手写，无法用鼠标点选。因此，你需要知道所使用的字体名称不是文件名。同时，你还需要理解字体顺序的规则：

  • 字体名称有空格的，要加上单引号
  • 前一字体失效，则采用后一字体
  • 加上通用字体

让我们以 VSCode 默认字体设置为例。打开设置，常用设置的第 3 项就是字体。注意：这里只是设置了编辑器的字体，VSCode 还可以单独设置别的面板的字体，可以在搜索栏里输入font字体关键字来过滤选项。

字体设置为：Consolas, 'Courier New', monospace。规则很简单：首先应用Consolas字体，如果使用了Consolas未包含的字符，则应用Courier New字体；如果还有未包含字符，则应用monospace字体。总而言之，如果Consolas能够包含所有的字符，那么整个编辑器文本就会全部应用Consolas字体。

小知识

字体的两个基本常识：

  • 编程字体通常不包含中文字符
  • 中文字体通常不是等宽的，不适合写代码

基于上述常识，我们应该把编程字体写在最前面，中文字体写在后面，最后再加上一个通用字体这是一种字体设置规范。比如本人 VSCode 的字体设置就是： 

        
          MonoLisa, '思源黑体 CN', monospace

特别提醒：MonoLisa、思源黑体 CN，都需要自己下载、安装。操作系统默认是没有这些字体的。这就是有的人把我的字体设置直接复制使用，却没有效果的原因。

这里给大家推荐两个安全字体安全就是指系统自带，无需下载。

  • Consolas，等宽字体，适合编程
  • 微软雅黑，适合中文，自从Windows Vista以来一直是系统默认字体

最后，我们来讨论一下通用字体，通用字体有 13 种：

比较常用的是serif衬线、sans-serif无衬线、monospace等宽。它们的区别，可以参考自定义字体专题。

括号配对着色

现代编程语言大多采用括号来分离层级关系，当一段程序有很多括号，意味着分很多层。如果所有括号都是同一种颜色，阅读起来会比较吃力，哪段程序属于哪一层，看得眼花缭乱。这时，同一层括号使用同一种颜色，不同层的括号使用不同颜色，能有效帮助我们阅读代码。

启用括号配对着色后，同级括号使用同种颜色，不同级括号使用不同颜色。原来需要安装扩展以提供此功能，现在已经内置到 VSCode 设置中。要打开这一功能，在设置搜索框里输入color pair过滤选项：

勾选Bracket Pair Colorization即可。

登录账号

VSCode 使用的最重要的 2 个账号：微软账号、GitHub 账号。

为什么要登录账号？

要先注册账号，才能登录账号。关于如何注册各类账号，请参阅注册重要账号。

登录账号，主要是为了同步设置。想象一下，你在一台电脑上花了很长时间，将 VSCode 的配置改好了；然后换了一台电脑，还要再重新设置一遍，是不是很麻烦？使用账号同步设置，换电脑之后只要登录同一个账号，就可以直接将设置同步过来，省时省力。

点击帐户按钮，按照指示操作。强烈建议两个账号都登录上去。登录完成之后，可以再次点击帐户按钮进行检查：

只要设置同步已打开已勾选，并且 GitHub 和 微软Microsoft账号已登录，就可以了。现在不管更改什么设置，都会自动同步到相关账号；换电脑只要再次登录账号即可。

更换主题

VSCode 拥有海量的主题可选。其主题主要分为 3 类：

  • 颜色主题
  • 文件图标主题
  • 产品图标主题

颜色主题指的是编辑器背景色和文本的配色。点击设置，选择主题中的颜色主题：

在弹出的颜色主题对话框中，使用 方向键来预览主题只是预览，不会直接应用。按Enter应用选中的主题，按ESC回到原来的主题。

提醒

如果对现有主题均不满意，可以将主题对话框滚到最上方，点选浏览其他主题…：

VSCode 会在服务器上搜索所有可用的主题，文件图标主题和产品图标主题也可以这样操作。如果最上方找不到此选项，就到最下方找。

文件图标主题指的是资源管理器文件列表的图标。点击设置，选择主题中的颜色图标主题：

优秀的文件图标能清晰表达文件的类型。

产品图标主题指的是各个面板上的产品图标样式。点击设置，选择主题中的产品图标主题：

可以选择自己喜欢的产品图标。

安装扩展

快捷键

VSCode 提供了极为丰富的快捷键。想查看所有快捷键，点击键盘快捷方式：

在设置菜单内选择键盘快捷方式以打开快捷键设置界面：

在这个界面中，你可以查看、修改、删除相关快捷键。VSCode 的快捷键实在是太多了，实际上我们常用的就几个，因此掌握常用快捷键即可。

  • 打开命令对话框Ctrl + Shift + P

    这个快捷键使用的频率非常高，因为很多命令都要在这个对话框中输入，比如重载窗口。

    按下快捷键后，弹出命令对话框，会列出所有的命令。输入关键词过滤一下，可以快速找到自己想要的命令。

  • 快速复制光标所在行Alt + Shift + 上下箭头

    如果想要复制一整行，用鼠标框选再复制、粘贴，实在太慢了，我们要活用编辑器提供的快捷键：

    无需用鼠标框选，只要将光标点到这一行任意位置即可。用 方向键来控制复制方向。

  • 格式化Alt + Shift + F

    平时书写代码时，少许空格、缩进不是很整齐，不用一个个改，直接按下快捷键，让编辑器进行格式化是最有效的办法：

  • 垂直选择Ctrl + Alt + Shift + 方向键

    垂直选择在需要添加、修改、删除大量相同的代码时非常有用：

    先点击起始处，再使用上下箭头来拉出一条直线。直线所在处可以统一操作。注意：这个快捷键也可以用左右箭头，这样就可以框选一个垂直的范围了。

    如果要框选一个垂直的范围，而不是直线，最快捷的方式是用快捷键配合鼠标左键进行点击。

    这个操作也可以用纯键盘完成，但因为框选的是一个范围，因此配合鼠标效率更高。

  • 打开设置窗口Ctrl + 逗号

    设置窗口也比较常用，每次都用鼠标效率太低，用快捷键最方便：

    使用此快捷键可以很方便地打开设置窗口。

提醒

不同软件的快捷键是会冲突的。比如我在一个机房就曾经遇到Ctrl + 逗号无效的情况，后来才发现这个快捷键被搜狗输入法占用了！当我把搜狗输入法的快捷键设置删除后，一切恢复正常。

导出和导入配置文件

何时需要导出和导入配置文件？

在2-7登录账号一节中，我们提到，如果登录账号，就可以在账号之间同步设置。但是，在某些场景下，我们不希望登录账号，或者无法登录账号：

  • 没有网络
  • 使用的是公共计算机，如：机房、网吧
  • 使用的是别人的私人电脑

如果想在这种情况下同步设置，就需要将设置导出，并在另一台电脑中导入。这样，在不登录的前提下，我们也可以在两台电脑之间快速应用相同的设置。

导出配置文件：

  1. 点选设置，选择导出配置文件
  2. 选择导出设置项

    可以自行选择导出哪些设置，没有特殊情况可以全选。

  3. 为配置文件命名
  4. 选择导出形式

    有两种导出形式可选，既然我们要纯离线运行，那就导出为本地文件。

  5. 选择导出位置

    选择导出文件的保存位置，点击保存即可。

  6. 导出成功提示

    成功导出后，会出现这样一个提示对话框。

  7. 查看导出文件

    导出的配置文件扩展名为.code-profile。

导入配置文件：

  1. 点选设置，选择导入配置文件
  2. 选择配置文件模板

    有多种配置文件模板可选，我们点击选择文件。

  3. 点选配置文件

    点击我们之前导出的配置文件，点击打开。

  4. 选择导入的配置项目

    与导出一样，你也可以选择导入哪些项目，选好后点击创建配置文件。注意：扩展越多，导入所需时间就越长。

  5. 选择导出位置

    耐心等待导入过程结束。

  6. 查看导入结果

    成功导入后，再次查看配置文件，此时名称已经变成了配置文件名，大功告成！
