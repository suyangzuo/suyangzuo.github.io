线性变换与仿射变换

线性变换

所谓线性变换，即参数与结果的变化率相等，。举个例子：函数y=f(x)，x增加n倍，y也增加n倍，这就叫线性函数，即f(n*x)=n*f(x)，映射到坐标轴中表现为一条直线，因此叫线性变换。

你会发现无论怎么改变参数，直线永远经过原点，即f(0*x)=0*f(x)，这是线性变换的一个重要特征：参数为0时，结果也为0。因此，我们可以总结一下线性变换的特征：

  1. 直线
  2. 经过原点

想一想，在遵守这两个特征的前提下，我们可以在几何空间中做哪些变换呢？缩放、旋转，对吧？但是无法移动，因为一旦移动，直线就不经过原点了；可是移动又是一个很常见的操作，怎么办呢？此时，我们就需要仿射变换了。

仿射变换

仿射，含义为相似的空间映射，它和线性变换基本一致，唯一的区别就是空间被平移了，其公式可以表示为：y=f(x)+b，这个b代表平移，当x=0时，y=b，因此直线不经过原点。

Canvas中使用的就是仿射变换。我们在各种图形软件中，看到的缩放视图、移动视图等功能，就是使用仿射变换来实现的。

Canvas变换

Canvas变换类型

在Canvas中，变换主要有：平移、旋转、缩放这3种，分别对应以下方法：

  1. 平移：ctx.translate(水平平移,垂直平移)
  2. 旋转：ctx.rotate(旋转弧度)
  3. 缩放：ctx.scale(水平缩放,垂直缩放)

相信你注意到了，所有的变换，都是应用在上下文，而不是某个图形，因此，对Canvas应用变换，如同现实中直接移动、旋转、放大黑板，黑板上画的所有图形都会受到影响。

Canvas变换累积

Canvas变换具有累积性，举个例子：运行ctx.scale(2,2)，整个Canvas视图会放大2倍，此时再次运行ctx.scale(2,2)，则会在放大2倍的基础上再次放大2倍，结果是一共放大了2×2=4倍。

不仅如此，假设先用ctx.rotate(Math.PI/4)顺时针旋转45°，再用ctx.translate(100,0)进行水平平移，则图形不会向右水平移动，而是会向右下方斜着移动，因为整个坐标轴的x轴已经被旋转了45°。

这种累积性往往不是我们想要的，为了避免累积，我们需要先重置变换，再应用新的变换。

      
        ctx.translate(旧水平平移, 旧垂直平移);
        ctx.rotate(旧弧度);
        ctx.scale(旧水平缩放, 旧垂直缩放);

        //方法1
        ctx.resetTransform(); //重置所有变换，推荐

        //方法2
        ctx.transform(1, 0, 0, 1, 0, 0); //重置所有变换

        ctx.translate(新水平平移, 新垂直平移);
        ctx.rotate(新弧度);
        ctx.scale(新水平缩放, 新垂直缩放);

很不方便是不是？别担心，我们马上就会讲到怎么解决这个问题。

Canvas变换顺序

平移、旋转、缩放，应用的顺序不同，最终效果也不同。一般来说，按照以下顺序更加符合人的期待：

  1. 平移
  2. 旋转
  3. 缩放

不同的变换顺序，得到的结果是不同的。

一次性应用全部Canvas变换

如前所述，Canvas变换具有累积性、顺序性，如果程序中需要经常应用各种变换，写代码会很累，因此，我们应该使用ctx.setTransform(a,b,c,d,e,f)函数，来代替平移、旋转、缩放这3个函数。

ctx.setTransform(a,b,c,d,e,f)具有以下优点：

  • 没有累积性：每次运行此函数，都会自动先重置变换。
  • 执行顺序符合人的心理预期：会自动按照平移、旋转、缩放的顺序执行。

ctx.setTransform(a,b,c,d,e,f)的参数含义如下：

  • a：水平缩放
  • b：水平倾斜
  • c：垂直倾斜
  • d：垂直缩放
  • e：水平平移
  • f：垂直平移

你可能觉得奇怪，怎么不是旋转，而是倾斜呢？因为Canvas变换是没有旋转参数的，所谓的旋转，其实就是用倾斜来实现的。

平移

平移相当于Photoshop中的移动视图，视图内的所有图形都会同步移动，以保持和原点的相对位置不变。

倾斜

单独的倾斜操作很少用到，倾斜的主要用途是实现旋转。试试将两个倾斜的数值设置为一正一负。

水平倾斜和垂直倾斜一正一负，就可以实现旋转，且图像的比例保持不变。但你肯定发现问题了：图像的尺寸变大了。为了实现旋转时尺寸保持不变，需要同时应用合适的缩放。

缩放

尝试负值，可以非常方便地实现水平翻转、垂直翻转等常见操作。

旋转

如前所述，Canvas变换是没有旋转参数的，旋转是用倾斜实现的，但是会导致图像变大，因此我们要同步应用合适的缩放来保持图像尺寸不变。

提醒

Canvas旋转的锚点永远都是原点，也就是世界坐标系的0,0。如果希望一个图形围绕自身旋转，就必须让图形的中心点位于原点。

提醒

为了保证在不同分辨率缩放的显示器上显示效果一致，Canvas的上下文一般都要进行缩放处理；对图像进行旋转时，又要同步应用缩放。然而，当使用setTransform时会先重置变换。也就是说，为图像旋转应用的缩放，会覆盖之前为了分辨率缩放应用的缩放。因此，为了保证旋转图像时尺寸不变，且分辨率缩放也维持正常，水平缩放和垂直缩放都要叠加dpr。 

        
          const dpr = window.devicePixelRatio;
          const 旋转弧度 = Math.PI / 4; //顺时针旋转45°
          const a = Math.cos(旋转弧度) * dpr;
          const b = Math.sin(旋转弧度);
          const c = -b;
          const d = a; //保证水平缩放和垂直缩放一致，比例不会失调
          const e = 0;
          const f = 0;
          ctx.setTransform(a, b, c, d, e, f);
