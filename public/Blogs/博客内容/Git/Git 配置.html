使用命令进行Git配置

提醒

Git 安装之后，第一件事就是配置用户名和邮箱，否则无法进行提交。

--global代表全局设置，如果不加这个参数，则设置仅应用到当前项目。

配置用户名

在终端运行如下命令：

git config --global user.name "用户名"

用户名可以跟 GitHub 用户名不一样。

配置用户邮箱

在终端运行如下命令：

git config --global user.email "用户邮箱"

邮箱一定要和代码托管网站邮箱一致如GitHub、Gitee。因此，GitHub和Gitee或是其它的代码托管网站最好使用相同的注册邮箱。

配置大小写敏感

Git 有一个忽略大小写配置，假设为true，则对文件夹和文件重命名时，如果仅仅只是修改大小写，Git 会认为此文件夹或文件没有被修改，这会造成严重的潜在问题！因此，强烈建议将此配置改成false， 

git config --global core.ignoreCase false

配置代理

GitHub在大陆地区难以访问，经常会遇到代码无法推送或拉取的情况，如果你使用了VPN，需要设置Git将本地计算机作为代理。在终端运行如下代码： 

      
        git config --global http.proxy 127.0.0.1:端口
        git config --global https.proxy 127.0.0.1:端口

提醒

代理端口要在代理软件中查看。我用的代理软件是Clash Verge，在不同操作系统下查看界面也不一样。

这是Clash Verge的Windows版本，使用的端口是7890。

这是Clash Verge的Linux版本，使用的端口是7897。

设置代理后，必须开启 VPN 才能跟 GitHub 通信。如果你的网络不需要 VPN 也能正常推送和拉取，那么你需要取消代理。在终端运行如下代码： 


      git config --global --unset http.proxy
      git config --global --unset https.proxy

运行上述代码之后，Git将不再使用代理。

提醒

--global 代表全局设置，只需设置一次，所有 Git 仓库的提交都会自动使用这个配置。如果不使用这个参数，则配置仅对当前仓库有效。

查看配置

查看用户名配置

在终端运行如下命令：

      
        git config user.name

查看邮箱配置

在终端运行如下命令：

      
        git config user.email

查看大小写敏感

在终端运行如下命令：

      
        git config core.ignoreCase

查看代理

在终端运行如下命令：

      
        git config http.proxy
        git config https.proxy

提醒

如果运行查看命令后，结果是空的，说明没有相关配置。
