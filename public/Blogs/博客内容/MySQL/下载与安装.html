Microsoft Visual C++ 可再发行程序包

Visual C++ 可再发行程序包简介

安装MySQL时，可能会遇到如下错误：

这是因为系统没有安装Microsoft Visual C++ 可再发行程序包。很多软件和游戏都要用到可再发行程序包。大多数情况下，可再发行程序包都会被自动安装。比如在 Steam 平台上，你会发现第一次运行某个游戏，会自动安装一些东西，其中就有可再发行程序包。然而MySQL不会自动安装可再发行程序包，只能自己下载安装。

Visual C++ 可再发行程序包有 3 个版本：

  • X86：32位版本
  • X64：64位版本
  • ARM64：移动64位CPU版本如：高通骁龙

一般情况下安装X86和X64版本即可。

微软提供了两个Visual C++ 可再发行程序包官方下载页面：

  • Microsoft Learn
  • Visual Studio 下载页

下载MySQL

MySQL版本介绍

MySQL面向不同商业用途，分成多个版本：

  • 经典版 Classic
  • 标准版 Standard
  • 企业版 Enterprise
  • 集群版 Cluster

这些版本的详细区别，可以参考大型商业客户版本列表和嵌入式数据库客户版本列表。然而，这些版本有一个共同点：要钱。

因此，很多人包括我在使用MySQL时，都会选择免费的社区版。

下载MySQL社区版

访问MySQL 官方网站：

点击DOWNLOADS，进入下载页：

点击MySQL Community Downloads社区版下载。

这里罗列了社区版的所有可用的产品。实际上，MySQL 社区版最核心的产品是：MySQL Community ServerMySQL 社区版服务器，它会提供MySQL的核心服务。不过，我们还是选择MySQL Installer for WindowsMySQL 安装包，因为它提供了一个直观的安装界面，操作起来很方便，而且它也包含了很多其它的产品，可以自由选择。

选择最新的版本，以及对应的操作系统。网络版和离线版，更加推荐离线版。因为网络版事实上还是会下载整个安装包，只不过是在安装过程中自动下载而已。离线版则可以彻底脱机运行，而且体积也不大。选好后，点击对应的Download。

MySQL会让你登录或者注册账号。如果没有这方面的需求，就直接点击No thanks, just start my download.

下载完成后，会得到一个安装包。

安装MySQL

选择安装类型

双击安装包进入安装界面：

一共有 4 种安装类型：

  • Server only 仅安装服务器
  • Client only 仅安装客户端
  • Full 完整安装
  • Custom 自定义

完整安装就是安装所有的产品。选好后点击Next。

安装产品

完整安装包含了 6 个产品：

  • MySQL Server 核心服务
  • MySQL Workbench MySQL官方图形客户端
  • MySQL Shell MySQL官方命令行工具
  • MySQL Router InnoDB集群的一部分，一种轻量级中间件
  • MySQL Documentation MySQL官方文档
  • Samples and Examples 样本与示例

作为核心服务，MySQL Server必装，其它的如需自选，请在第 1 步选择Custom。确认无误后，点击Execute执行, 等待所有产品安装完毕：

当所有产品前面都显示✅，代表安装完成，点击Next进入产品配置。

产品配置：MySQL Server服务器

  1. 准备配置MySQL Server服务器

    有 3 个产品需要进行配置，配置向导会带着你一步步完成。点击Next进行MySQL Server配置：

  2. 类型与网络

    这里主要是选择端口号和配置类型。端口号默认3306，无特殊需求不用改。

    配置类型分为 3 种：

    • Development Computer开发机：做开发用的电脑，会安装运行大量软件，MySQL只占用少许内存。
    • Server Computer服务器：电脑上会运行几个别的程序，如果电脑是网络服务器或者应用服务器，可以选这个。MySQL占用中等量内存。
    • Dedicated Computer专用服务器：电脑上只运行MySQL 服务，MySQL会占用所有可用内存。

    根据实际情况选择，我们的电脑是私人电脑，因此选择Development Computer。选好后点击Next。

  3. 验证方法

    身份验证的方式分为两种：

    • Use Strong Password Encryption for Authentication (RECOMMANDED)推荐：使用基于SHA-256的密码加密模式，不兼容MySQL 5.x 版本。
    • Use Legacy Authentication Method (Retain MySQL 5.x Compatibility)：使用传统验证方法，兼容MySQL 5.x 版本。

    提醒

    加密模式是可以修改的，因此选错了也不要紧。出于安全考虑，肯定是选择推荐模式比较好。不过，个别数据库客户端只支持老的验证方法，用推荐模式会导致无法登录。

    确定验证方法后，点击Next。

  4. 账户与角色

    为root账户设置密码，两次密码要一致方可。

    提醒

    root是MySQL数据库最重要、权限最大、对安全性要求最高的账户，因此在实际工作过程中，一定要为该账户设置足够复杂的密码！像图 18中的密码就不符合要求，因为出现了Weak弱提示！实际工作请务必使用Strong强密码！

MySQL服务与环境变量

MySQL服务

在3-7中，我们将MySQL Server配置为Windows 服务名称为MySQL80。现在，可以打开服务管理器，来查看这个服务是否正常运行。要查看服务，有两种方式：

  • 服务窗口

    这是一种比较传统的方式。按Win + R快捷键，打开运行对话框，输入命令：

    运行services.msc即可打开服务窗口：

    在服务列表里找到MySQL80，显示其正在运行，说明服务器正常。

  • 任务管理器中的服务

    这是较新的方式。按Ctrl + Shift + ESC快捷键，打开任务管理器，点击服务：

    在服务列表里找到MySQL80，显示其正在运行，说明服务器正常。

MySQL环境变量

在使用MySQL的过程中，我们需要经常运行mysql.exe、mysqldump.exe等程序，因此我们需要将它们的路径加入环境变量：

MySQL的默认安装路径是：C:\Program Files\MySQL，因此它的服务器相关程序的默认路径是：C:\Program Files\MySQL\MySQL Server 8.0\bin，将该路径加入环境变量即可。
