Linux 软件生态

派系之争

开放、自由是Linux的核心特征，也是一把双刃剑。它既给了用户最大的自由与选择权，也导致了生态的碎片化，这一点在软件包领域体现得尤为明显。

例如，三大派系分别有自己的包管理器：

派系 底层工具 包管理器 包格式 依赖解决
Debian dpkg apt .deb 自动
红帽 rpm dnf .rpm 自动
Arch pacman pacman .pkg.tar.zst 自动

一个派系下的包管理器，无法在另一个派系下使用真要用也有办法，但是很麻烦，这种分裂的生态，导致用户的学习成本与迁移成本都大幅增加。为了解决这个问题，各个社区、公司都在研究通用软件包格式。

通用软件包格式

戏剧性的是，通用软件包格式依然有派系之争！每个派系都希望自己说了算，真是让人心累呀！

通用包格式 开发人员 所属派系
AppImage Simon Peter 无
Flatpak Alexander Larsson 红帽
Snap Canonical 公司 Debian

定睛一看，竟然还有一个无派系的！没错，AppImage也是我个人最喜欢的包格式。但是，这些通用包格式依然区分派系的结果就是：没有哪个包格式能够一统天下。

通用包格式虽然也有派系之争，但是这些包格式在所有的发行版中都可以正常安装、使用。

AppImage

安装libfuse2

AppImage文件需要root用户权限进行挂载，而libfuse2允许非root用户进行挂载。

在有的系统上，包名称可能叫libfuse2t64。

下载运行AppImage

AppImage是我最喜欢的通用包格式，它最大的优势在于单文件、全依赖包含，因此下载AppImage是最简单的事情。

Flatpak

安装Flatpak

要使用Flatpak软件包，需要先安装Flatpak自身。可以访问Flatpak 官方网站查看安装方法。

不同发行版的安装方式各有不同。只有先安装Flatpak自身，才能安装Flatpak软件包。

添加Flathub软件仓库

Flatpak软件仓库是一个很长的URL地址，因此需要给这些URL起一个别名。目前最流行的仓库就是Flathub。 

      
        //flatpak 添加远程仓库 如果已经存在就算了 别名 URL地址
        flatpak remote-add --if-not-exists flathub https://dl.flathub.org/repo/flathub.flatpakrepo

修改Flathub软件仓库

默认情况下，Flathub仓库的URL地址是国外的，从国内访问速度很慢，因此需要修改为国内的镜像地址。

  • 上海交通大学：https://mirror.sjtu.edu.cn/flathub
  • 中国科学技术大学：https://mirrors.ustc.edu.cn/flathub 
      
        //sudo flatpak 修改远程仓库 别名 --url=URL地址
        sudo flatpak remote-modify flathub --url=https://mirrors.ustc.edu.cn/flathub

查看Flathub软件仓库

      
        //查看远程仓库别名
        flatpak remotes

flatpak remotes可以查看仓库别名，我只有一个仓库，别名为flathub。但是这个命令看不到仓库的URL地址，我们需要加一个-d参数。 

      
        //查看远程仓库详细信息
        flatpak remotes -d

搜索Flatpak软件包

想要安装Flatpak软件包，需要知道这个软件包的ID。 

      
        //flatpak 搜索 软件名称
        flatpak search krita

Krita的ID是org.kde.krita。

安装Flatpak软件包

有了ID，就可以进行安装了。

      
        //flatpak 安装 仓库别名 软件ID
        flatpak install flathub org.kde.krita

安装过程中会询问继续对系统安装进行这些更改吗？输入y，回车，耐心等待安装完成。完成后就可以在应用列表里看到Krita了。

除了用命令搜索软件包ID，还可以访问Flathub 官方网站直接获取安装命令。

在软件包的页面中，点击安装旁边的倒三角，就可以看到写好的安装命令，直接复制到终端执行即可。

查看Flatpak可更新的软件包

想查看已安装且有更新的Flatpak软件包，用flatpak remote-ls --updates命令。 

      
        //flatpak 远程-列表 --有更新
        flatpak remote-ls --updates

命令运行后列出的就是已经安装且可更新的软件包。

更新Flatpak软件包

想一口气更新所有的Flatpak软件包，用flatpak update命令。 

      
        //flatpak 更新
        flatpak update

无事可做说明所有软件都是最新的，不需要更新。如果想更新某个特定软件，给命令加上ID即可： 

      
        //flatpak 更新 软件ID 
        flatpak update org.kde.krita

查看Flatpak已安装的软件包

想知道自己用Flatpak安装了哪些软件？用flatpak list即可！ 

      
        //flatpak 列表
        flatpak list

很多软件都会附带其它的包，因此会列出很多很多个软件包。

卸载Flatpak软件包

卸载Flatpak软件包，同样要用到ID。 

      
        //flatpak 卸载 软件ID
        flatpak uninstall org.kde.krita

卸载过程中会询问继续对系统安装进行这些更改吗？输入y，回车，耐心等待卸载完成。

Snap

安装snapd守护进程

要使用Snap软件包，需要先安装snapd 守护进程。部分Linux发行版已经预装了snapd。可以访问Canonical Snapcraft 官方文档查看安装方法。

部分Linux发行版已经预装了snapd，其它发行版的安装方式各有不同。只有先安装snapd守护进程，才能使用snap软件包。

搜索Snap软件包

想要安装Snap软件包，需要知道这个软件包的名称。 

      
        //snap 寻找 软件名称
        snap find krita

注意：名称不能想当然！Krita的名称就叫Krita，但不是每个软件都这么简单直接，比如Visual Studio Code的名称是code。因此，名称要以搜索结果为准，不能凭想象。

安装Snap软件包

有了软件名称，就可以进行安装了。

      
        //snap 安装 软件名称
        snap install krita //无论加不加 sudo，都会让你输入密码，只不过输入的界面不同

查看Snap已安装的软件包

想知道自己用Snap安装了哪些软件？用snap list即可！ 

      
        //snap 列表
        snap list

查看Snap可更新的软件包

想查看已安装且有更新的Snap软件包，用snap refresh --list命令。 

      
        //snap 刷新 --列表
        snap refresh --list

命令运行后列出的就是已经安装且可更新的软件包。

更新Snap软件包

想一口气更新所有的Snap软件包，用snap refresh命令。 

      
        //snap 刷新
        snap refresh

想更新特定的Snap软件包，给命令加上软件名称即可。 

      
        //snap 刷新 软件包名称
        snap refresh krita

卸载Snap软件包

卸载Flatpak软件包，同样要用到软件名称。 

      
        //snap 移除 软件名称
        snap remove krita
