渐变的类型

提醒

如果你熟悉Photoshop，完全可以把渐变的代码想象成Photoshop渐变编辑器中的滑块，代码与滑块的本质是完全一样的，这可以更加方便地帮助你理解渐变。

从Photoshop 2023开始，Adobe提供了全新的渐变工具，能够实现更加直观、实时的渐变编辑。而原来的渐变编辑器变成了经典渐变。

本博客后续的内容中，我会根据需要使用渐变或经典渐变的截图。

线性渐变

线性渐变是最常用的渐变，所谓线性，代表颜色变化的方向是一条直线。

径向渐变放射渐变

径向代表颜色变化的方向是从中心点向四周扩散。

角度渐变圆锥渐变

角度代表颜色变化的方向是环绕着中心点旋转。

CSS线性渐变方向之坑

如果我们希望渐变的方向是水平的，应该怎么做？

在Photoshop中，将渐变的方向顺时针旋转 90° 即可。在CSS代码中，可以通过添加旋转角度达到同样的效果。 

      
        /* background-image: linear-gradient(顺时针旋转角度, 颜色 色标位置, 颜色 色标位置); */
        background-image: linear-gradient(90deg, black 0%, white 100%);
我是代码，不是图片

奇怪，怎么反过来了呢？不熟悉CSS线性渐变的人可能会摸不着头脑，怎么调整都得不到想要的结果。让我们来看看MDN 官方线性渐变页面的解释：

CSS线性渐变的默认方向是to bottom，而to bottom又等价于180deg！简而言之，CSS线性渐变会默认旋转180°！也就是说，上黑下白的渐变，其原本的效果旋转角度为0应该是上白下黑，再顺时针旋转90deg，就变成左黑右白了。

知道了CSS线性渐变默认旋转180°后，修改代码就变得简单了。既可以将前后颜色顺序互换，也可以逆时针旋转 90°或顺时针旋转270°。 

      
        /* background-image: linear-gradient(顺时针旋转角度, 颜色 色标位置, 颜色 色标位置); */
        background-image: linear-gradient(-90deg, black 0%, white 100%);
        
        /* 或者 */
        background-image: linear-gradient(90deg, white 0%, black 100%);
我是代码，不是图片

这回就得到了正确的效果。记住：当完全不旋转设置旋转角度为0deg时，后面的颜色会在上方。

区间内颜色不变

如果希望渐变有一段区间的颜色保持不变，该怎么做呢？比如，我希望从渐变的1/3到2/3都是绿色：

答案很简单：在1/3处和2/3处各放置一个绿色色标即可。对应的代码如下： 

      
        /* ------------------------
        background-image: linear-gradient(
          顺时针旋转角度, 
          黄色 黄色标位置,
          绿色 绿色标位置,
          绿色 绿色标位置,
          紫色 紫色标位置
        );
        --------------------------- */
        background-image: linear-gradient(
          90deg,
          yellow 0%,
          green 33.33%,
          green 66.67%,
          purple 100%
        );
我是代码，不是图片

如上所示，在相应区间的起点和终点，各放置一个关键帧，这个渐变从1/3处到2/3处都保持绿色不变。

控制颜色的过渡距离

色标之间的距离越远，颜色过渡就越平滑；反之离得越近，过渡就越急促。当两个色标的位置相同时，颜色会瞬间完成过渡，变成一条线，仿佛颜色之间发生了碰撞，形成独特的撞色效果。

两个颜色快彼此紧邻，能够营造出撞色效果。

      
        /* ------------------------
        background-image: linear-gradient(颜色1 色标位置, 颜色2 同样的色标位置);
        --------------------------- */
        background-image: linear-gradient(90deg, black 50%, white 50%);

径向渐变放射渐变

径向渐变的本质

径向渐变的本质就是三个属性：

  • 圆心位置
    • 水平默认50%
    • 垂直默认50%
  • 渐变结束时的形状
    • ellipse椭圆默认
    • circle正圆
  • 结束形状的尺寸
    • px、%等各种长度单位
    • closest-side最近边
    • closest-corner最近角
    • farthest-side最远边
    • farthest-corner最远角默认

径向渐变默认属性

让我们先用默认的属性以及黑白色创建一个径向渐变：

默认属性如下：

  • 圆心位置：center正中间
  • 渐变结束时的形状：ellipse椭圆
  • 结束形状的尺寸：farthest-corner最远角，结束形状与离圆心最远的角相交

将这些属性写成CSS径向渐变代码： 

      
        /* --------- 省略型写法：--------- */
        background-image: radial-gradient(黑色 黑色标位置, 白色 白色标位置);

        /* --------- 完整型写法：--------- */
        background-image: radial-gradient(
          形状 尺寸 at 圆心位置, 
          黑色 黑色标位置, 
          白色 白色标位置
        );
        
        /* --------- 完整型写法实际代码如下：--------- */
        background-image: radial-gradient(
          ellipse farthest-corner at center, 
          black 0%, 
          white 100%
        );

这会创建一个由黑到白的椭圆渐变，圆心在正中间，椭圆的尺寸正好与离圆心最远的角相交；渐变的起点在圆心，终点在椭圆的边缘。

我是代码，不是图片

径向渐变自定义属性

椭圆，意味着水平半径和垂直半径的尺寸可以不同。因此，椭圆的尺寸可以用两个值来表示，分别代表水平半径和垂直半径。 

      
        /* --------- 省略型写法：--------- */
        /* --------- 圆心默认在正中间--------- */
        background-image: radial-gradient(
          水平半径 垂直半径, 
          黑色 黑色标位置, 
          白色 白色标位置
        );
        
        /* --------- 完整型写法：--------- */
        background-image: radial-gradient(
          椭圆 水平半径 垂直半径 at 圆心位置, 
          黑色 黑色标位置, 
          白色 白色标位置
        );
        
        /* --------- 完整型写法实际代码如下：--------- */
        background-image: radial-gradient(
          ellipse 300px 50px at center,
          black 0%, 
          white 100%
        );
我是代码，不是图片

注意：要让水平半径和垂直半径不同，必须使用长度单位，不能用closest-corner这种关键词。

现在，让我们来试试修改圆心位置、形状、尺寸，并且从内到外形成三个不同颜色的环状。

目测一下关键属性：

  • 圆心位置：水平大约位于90%，垂直大约位于不到30%
  • 渐变结束时的形状：circle正圆
  • 结束形状的尺寸：farthest-side最远边，结束形状与离圆心最远的边相交
  • 色标位置：每两种颜色之间的过渡，大约占半径的1/3。
  • 注意：两种颜色彼此碰撞，必然有两个色标彼此非常接近，甚至位于同一位置。因此，黄红交界处、红绿交界处，都存在两个色标。图11很难看出来，因为色标之间挨得特别近。 
      
        /* --------- 语法：--------- */
        background-image: radial-gradient(
          正圆 最远的边 at 圆心水平位置 圆心垂直位置, 
          绿色色标 1/3,
          红色色标 1/3,
          红色色标 2/3,
          黄色色标 2/3
        );
        
        /* --------- 完整型写法实际代码如下：--------- */
        background-image: radial-gradient(
          circle farthest-side at 90% 27%,
          seagreen 33.33%,
          indianred 33.33%,
          indianred 66.67%,
          darkgoldenrod 66.67%
        );
我是代码，不是图片

再次强调：只有两个色标的位置非常接近，甚至相同，才会出现撞色的效果。

角度渐变圆锥渐变

把它想象成圆锥

设置一个圆心，绕它一圈，这是什么？圆锥！

从Photoshop的角度来看，角度渐变需要什么？

  • 起始角度，默认为0，因此图中的线是笔直朝上的
  • 圆心位置：默认居中
  • 色标位置：可以是一个值起始位置，也可以是两个值起始位置和结束位置。图中一共有3个色标：
    • 绿色标，位于0deg等价于0%
    • 红色标，位于180deg等价于50%
    • 黄色标，位于360deg等价于100%

三个色标形成了两次过渡：绿 → 红，红 → 黄

      
        /* --------- 语法：--------- */
        background-image: conic-gradient(
          from 渐变顺时针旋转角度 at 圆心位置,
          颜色1 色标位置,
          颜色2 色标位置,
          颜色3 色标位置
        );
        
        /* --------- 省略型写法：--------- */
        background-image: conic-gradient(
          /* 不写颜色占比，所有颜色就会在渐变内平均分布 */
          seagreen,
          indianred,
          darkgoldenrod
        );

        /* --------- 完整型写法：--------- */
        background-image: conic-gradient(
          from 0deg at center,
          seagreen 0%,
          indianred 50%,
          darkgoldenrod 100%
        );
我是代码，不是图片

解读一下这个渐变：

  • 绿色从 0° 开始变化，到 180° 变成红色
  • 红色从 180° 开始变化，到 360° 变成黄色

完美的 360° 自然过渡

现在的渐变还有个问题，有一条笔直的线，仿佛黄色和绿色撞在一起。怎么才能去掉它呢？想想之前所学，撞色是怎么产生的？没错，是因为色标之间离得太近，我们把黄色标远离绿色标试试：

😕😕😕没用！😕😕😕这是因为黄色标是最后一个色标，而最后一个色标之后的空间会自动填充最后一种颜色，也就是黄色。假设黄色标位于 300°，那么 300° - 360° 都会自动填充黄色。最后一个色标和第一个色标之间是不会过渡的。

解决的办法是什么？很简单：将最后一个色标变成绿色。因此，我们需要再添加一个绿色标。

看上去已经很不错了，对不对？但如果我们想让三种颜色平均分布呢？仔细想想：

  • 本来只有2次过渡：
    • 绿 → 红
    • 红 → 黄
  • 现在有3次过渡且需要平均分布：
    • 绿 → 红
    • 红 → 黄
    • 黄 → 绿

3次过渡，又要平均分布，那我们就要把绿、红、黄三个色标在 360° 范围内均匀摆放，并且最后一个绿色标和第一个绿色标的位置要重叠：

注意：0°和360°各有一个绿色色标，视觉上彼此重叠，看起来像是一个色标。 

      
        /* --------- 语法：--------- */
        background-image: conic-gradient(
          from 渐变顺时针旋转角度 at 圆心位置,
          颜色1 色标位置,
          颜色2 色标位置,
          颜色3 色标位置,
          颜色1 色标位置 /* 此位置必须与颜色1的色标位置重叠，因此用 颜色1色标位置 + 360° */
        );
        
        /* --------- 代码：--------- */
        background-image: conic-gradient(
          from 0deg at center,
          seagreen 0deg,
          indianred 120deg,
          darkgoldenrod 240deg,
          seagreen 360deg
        );
我是代码，不是图片

角度渐变自定义属性

让我们来写个圆心位于偏右上角、起始角度为 60°、三色撞色角度渐变： 

      
        /* --------- 语法：--------- */
        background-image: conic-gradient(
          from 渐变顺时针旋转角度 at 圆心水平位置 圆心垂直位置,
          seagreen 起始角度 结束角度, /* 此处为相对角度 */
          indianred 起始角度 结束角度, /* 此处为相对角度 */
          darkgoldenrod 起始角度 结束角度 /* 此处为相对角度 */
        );
        
        /* --------- 代码：--------- */
        background-image: conic-gradient(
          from 60deg at 70% 25%,
          /* 色标的角度均为相对角度。渐变整体旋转了60°，此时渐变内的 0° 看上去就像是 60° 的斜线 */
          seagreen 0deg 120deg, /* 此处为相对角度 */
          indianred 120deg 240deg, /* 此处为相对角度 */
          darkgoldenrod 240deg 360deg /* 此处为相对角度 */
        );
