隐藏元素原创

作者： 苏扬

日期： 2024年1月27日

隐藏，不只是看不见

布局

被隐藏元素是否参与布局，这决定了该元素隐藏后，是否会影响其它元素的布局。

交互

被隐藏元素是否可以与用户交互，比如对鼠标操作是否有反应。

可见性

元素是否可见，这也是隐藏一个元素最常见的目的。

隐藏元素的方式

提醒

为了让效果更加可视化，我设置了一个伪类，只要将鼠标悬停在中间的盒子上，左右两个盒子的背景色就会变成深橄榄绿色。同时，为中盒子设置了 1 秒的过渡，用来观察各个属性的过渡效果。

display: none

display: none 属性影响
布局 交互 可见

display: none不仅仅是隐藏元素，而是直接取消了元素的布局，相当于此元素根本不存在，既然不存在，那也就无法交互了。同时也可以很清楚地看到，display属性是无法过渡的，元素会瞬间消失，瞬间出现。

visibility: hidden

visibility: hidden 属性影响
布局 交互 隐藏

visibility: hidden隐藏了元素，且无法交互；但元素所占的位置没有改变，这就意味着元素的布局依然是正常的。仔细观察的话，会发现visibility对于过渡是有反应的，它会等待 1 秒再消失，但还是瞬间消失，没有过渡效果，

opacity: 0

opacity: 0 属性影响
布局 交互 隐藏

opacity: 0将透明度设置为0使得元素不可见，并且依然可以和元素交互；同时元素的布局依然保持正常。可以注意到：opacity是可以过渡的。

scale: 0

scale: 0 属性影响
布局 交互 隐藏

scale: 0对元素进行缩放，缩小到0就看不见了，也不能再跟元素进行交互；但是元素的布局依然保持正常。scale是可以正常过渡的。
